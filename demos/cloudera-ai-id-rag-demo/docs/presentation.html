Retrieval Stack

3-Stage Retrieval for Maximum Accuracy

Each stage improves result quality. The cross-encoder final stage matches enterprise accuracy requirements for citation-grounded answers.

Stage 1 — Semantic Search
FAISS · multilingual-e5-large · 1024-dim · cosine similarity
↓ top-40 candidates
Stage 2 — Keyword Re-ranking
BM25 (rank-bm25) + FAISS → RRF fusion · parallel retrieval
↓ fused top-20
Stage 3 — Cross-Encoder Reranking
ms-marco-MiniLM-L-6-v2 · ~90 MB · lazy-loaded · thread-safe
↓ top-K chunks → LLM

Technical Deep Dive
Security & Observability

Enterprise-Grade Security Out of the Box

Security Controls
✓
SQL AST guardrails
sqlparse AST walking — blocks DROP/DELETE/ALTER even in CTEs and subqueries. Belt-and-suspenders SELECT guard at execution boundary.
✓
Vector store integrity
SHA-256 hash written after every FAISS build, verified before load. Detects tampering or corruption.
✓
XSS protection
All LLM output is sanitised with DOMPurify before dangerouslySetInnerHTML. No raw HTML from untrusted sources.
✓
Configure allowlist
POST /api/configure validates keys against an explicit allowlist. Platform env vars cannot be overwritten via the wizard.
✓
CML SSO / LDAP
Auth handled entirely by Cloudera AI Workbench — Remote-user headers injected. App implements no custom auth.
Observability
MLflow Integration
Latency, token usage, provider, mode — logged async per inference. Ring-buffer always on, MLflow optional via MLFLOW_TRACKING_URI.
Metrics Dashboard
Built-in /metrics page — session token counters, latency charts, per-run table, mode breakdown. Management-ready, no setup required.
Iceberg Time Travel
On CDW/Trino: query historical snapshots of Iceberg tables. Ask "what did the portfolio look like on Jan 1?" — auditor-grade capability.
