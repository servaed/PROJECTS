Intelligence Layer

Three-Mode Intelligent Routing

The router classifies every question in milliseconds using keyword heuristics — no LLM call unless genuinely ambiguous. History-aware for multi-turn conversations.

◈
Document RAG
Policy & Procedure Questions
"What are the credit restructuring conditions?"

Retrieves from 14 policy documents using hybrid BM25 + FAISS + cross-encoder reranking.
⬡
Structured Data
Aggregation & Listing Questions
"Show top 5 customers by credit exposure."

Generates and executes validated SQL with retry on failure. Shows live data from 9 tables, 1,485 rows.
⬡
Combined
Comparison & Compliance Questions
"Has MSME credit outstanding met the 15% expansion policy target?"

Runs both retrieval and SQL in parallel, then synthesises a single grounded answer.
⚡ 4-tier heuristics — no LLM call for unambiguous questions
🕐 History-aware — follow-ups like "how about telco?" resolve correctly
⟳ SQL retry loop — auto-rephrases on generation failure

Value Proposition

Enterprise Challenges Solved

✓
Data silos between documents and databases
Policy lives in PDFs; facts live in SQL tables. This demo bridges both in a single answer — with sources cited for each.
✓
Hallucination in LLM answers
Every statement is grounded to a retrieved chunk or SQL result. The grounding confidence badge signals whether the answer has sources.
✓
Language barriers in Indonesian enterprise
Answers in the user's language — Bahasa Indonesia or English — detected per message. Multilingual embeddings support both natively.
✓
Demo portability and setup overhead
One command (make demo-reset) restores a clean state. Runs on any laptop with Python 3.10+ — no Docker, no cloud account needed.
✓
LLM vendor lock-in risk
Pluggable LLM clients for Cloudera CAII, OpenAI, Azure, AWS Bedrock, Anthropic, and local models (Ollama/vLLM). Switch with one env var.
✓
Knowledge base management in production
Upload docs via drag-and-drop, scrape URLs, or import CSV/Excel — all from the browser. Nightly re-ingestion runs as a CML Job automatically.
✓
Retrieval quality degradation at scale
Three-stage retrieval: FAISS semantic search → BM25 keyword re-ranking → cross-encoder reranking. Each layer catches what the previous misses.
✓
Observability and cost tracking
Every inference is logged to the /metrics dashboard and optionally to MLflow — provider, model, latency, token usage, mode distribution.

Cloudera Platform

Cloudera Platform Showcase

Every component maps to a Cloudera product or capability — making it a living reference architecture for customer conversations.

🤖
Cloudera AI Inference
The default LLM provider. Hosts Llama, Mistral, and other models on the customer's own CDP infrastructure. The LLM Compare panel benchmarks it against other providers live.

Cloudera AI Workbench (CML)
The entire demo deploys as a CML Application — one Git repo, one Script field, zero infrastructure setup. CML Jobs run nightly re-ingestion automatically.
🗄
CDW (Trino / Impala)

Parquet files are structured as Iceberg-compatible table data. The Time Travel tab demonstrates Iceberg's point-in-time query capability — a unique Cloudera differentiator.

MLflow Experiment Tracking
Logs every chat inference — provider, model, latency, token usage, mode — to a MLflow experiment. The /metrics dashboard shows live session analytics.

Technical Depth

Production-Grade RAG Quality Stack

1️⃣
FAISS Semantic Search
multilingual-e5-large · 1024-dim · cosine similarity
↓ fuse via RRF
2️⃣
BM25 Keyword Re-ranking
rank-bm25 · entity / code name recall
↓ Reciprocal Rank Fusion
3️⃣
Cross-encoder Reranking
ms-marco-MiniLM-L-6-v2 · joint (query, chunk) scoring
↓ top-K to LLM
✦
LLM Synthesis + Citations
streaming SSE · grounding badge · full chunk preview
✓
Prompt caching
Anthropic cache_control + Bedrock cachePoint on system prompts. Cuts cost ~90% and latency ~30% for repeated queries.
✓
SQL few-shot examples
6 worked domain examples in the generation prompt. Dramatically reduces TIDAK_DAPAT_DIJAWAB on edge queries.
✓
Agentic SQL retry
On generation failure, automatically rephrases the question and retries once — invisible to the user.
✓
86 unit tests · 36-question bilingual eval
Full test coverage including 10 multi-turn conversation scenarios across all three domains.
