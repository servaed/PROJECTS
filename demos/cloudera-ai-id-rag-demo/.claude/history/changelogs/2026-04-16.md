## P1: Bilingual Response Support

- `src/llm/prompts.py`:
  - System prompts now use `{lang_rule}` placeholder
  - `_lang_rule(language)` returns "Jawab selalu dalam Bahasa Indonesia..." or "Always respond in formal, concise English."
  - All `build_*` prompt functions accept `language: str = "id"` parameter
  - Added `get_answer_not_found(lang)` and `get_answer_sql_failed(lang)` for localised fallbacks
- `src/orchestration/answer_builder.py`:
  - `AnswerPrep.language: str = "id"` field added
  - `prepare_answer()` accepts and threads `language` into `AnswerPrep`
  - `_build_messages()` passes `language` to all prompt builders
  - `_get_fallback()` uses language-aware fallback strings
- `app/api.py`:
  - `_chat_sse()` accepts `language: str = "id"` parameter
  - Call site passes `body.language` to `_chat_sse`
  - `prepare_answer` call includes `language` positional arg
