## SQLite ‚Üí DuckDB Migration

Replaced the embedded SQLite + SQLAlchemy stack with DuckDB reading Parquet files.
This aligns the local dev query dialect with production Trino (both use standard SQL
on Parquet/Iceberg files) and removes the SQLAlchemy dependency.

- **`src/connectors/duckdb_adapter.py`** (new) ‚Äî DuckDB in-process engine; registers one
  view per `.parquet` file in `DUCKDB_PARQUET_DIR`; thread-safe singleton connection.
- **`src/connectors/db_adapter.py`** ‚Äî default engine changed from `"sqlite"` to `"duckdb"`;
  added `get_table_row_count()` and `get_engine_label()` factory helpers.
- **`data/sample_tables/seed_parquet.py`** (new) ‚Äî seeds 9 Parquet files from `sample_data.py`
  using pandas + pyarrow; replaces `seed_database.py`.
- **`data/sample_tables/seed_database.py`** ‚Äî deleted (replaced by `seed_parquet.py`).
- **`src/config/settings.py`** ‚Äî `query_engine` default changed to `"duckdb"`;
  added `duckdb_parquet_dir` field; removed `database_url`.
- **`requirements.txt`** ‚Äî added `duckdb>=1.1.0`, `python-multipart>=0.0.9`;
  removed `SQLAlchemy`.
- **`deployment/launch_app.sh`** ‚Äî step 3 now seeds Parquet files via `seed_parquet.py`
  instead of SQLite; checks for `msme_credit.parquet` sentinel file.
- **`Makefile`** ‚Äî `make seed` calls `seed_parquet.py`.
- **`CLAUDE.md`** ‚Äî updated all SQLite references to DuckDB/Parquet.
