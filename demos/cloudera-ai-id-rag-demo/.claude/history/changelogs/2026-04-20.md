# Changelog — 2026-04-20

## P1: Demo UX Seamlessness (9 improvements)

### LLM warm-up ping at startup
- `app/api.py` `lifespan()` spawns a daemon thread before `yield` that sends a 1-token
  ping to the LLM in a `ThreadPoolExecutor(max_workers=1)` with a 5-second timeout
- First real user query benefits from primed model cache; non-critical — timeout is logged
  but does not block startup

### Auto-play pause / resume
- Added `isAutoPaused` state + `autoPausedRef`, `pausedAtIdxRef`, `playNextRef` refs
- During auto-play the sidebar shows **⏸ Pause** and **⏹ Stop** side by side
- When paused, buttons swap to **▶ Resume** and **⏹ Stop**
- `resumeAutoPlay()` calls `playNextRef.current(pausedAtIdxRef.current)` to pick up exactly
  where it left off

### localStorage persistence (version 3)
- All `sessionStorage` references in `index.html` replaced with `localStorage`
- `_SS_VER` bumped 2 → 3 — stale v2 data is discarded on first load

### Total response latency badge
- `handleSubmit` records `var t0 = Date.now()` before SSE dispatch, stores
  `latency_ms: Date.now() - t0` in the DONE reducer case
- `AssistantMsg` renders `⚡ X.Xs` in green monospace when `msg.latency_ms != null`

### Full demo reset
- Sidebar footer: **↺ Reset Demo** button calls `onFullReset`
- `onFullReset`: stops auto-play, sets domain `banking`, language `id`, dispatches `RESET`,
  clears `localStorage`

### Keyboard shortcuts
- `Ctrl+Shift+D` — toggle demo auto-play (start or stop)
- `Ctrl+K` — clear conversation
- `Ctrl+Shift+R` — full reset (domain + language + history)
- `Escape` — stop auto-play if running

### Domain-aware welcome screen
- `Welcome` component shows domain icon + name + description + top 3 clickable sample prompts
- Clicking a prompt fires `handleSubmit` directly; domain switch updates prompts immediately

### Inline bar chart (DataChart)
- `DataChart` component parses markdown table (pipe-delimited), detects numeric columns,
  draws a Canvas bar chart with `ctx.roundRect()` rounded bars
- `SqlPanel` renders `<DataChart>` when `sc.row_count >= 2 && sc.row_count <= 12`

### First-launch setup overlay
- `SetupOverlay` component: full-screen overlay with 4-step setup instructions + link to
  `/configure` + "Continue anyway" dismiss button
- `/health` fetch in `App` effect checks `llm_configured === false` → shows overlay once
  per browser session (dismissed via `sessionStorage` key)

## P2: Deploy Anywhere / Configure Anywhere (10 improvements)

### 1. Makefile
- Targets: `help` (default), `dev`, `seed`, `reset-vs`, `test`, `test-fast`,
  `docker`, `docker-run`, `docker-push`, `clean`
- Variables: `REGISTRY`, `IMAGE`, `TAG`, `PORT`

### 2. docker-compose.yml
- Single-service compose: builds from `Dockerfile`, mounts named volumes
  `vector-store` and `minio-data`, publishes `:8080`
- `env_file: [.env]` with `required: false` — no error if `.env` absent
- Healthcheck on `/health` with `start_period: 300s`, resource limits `4CPU / 8G`

### 3. GitHub Actions CI/CD
- `.github/workflows/docker-build.yml`: pushes to GHCR on push to `main`/`master`
  or semver tags; no push for PRs (build only)
- Tags: branch name, semver `{{version}}`, `{{major}}.{{minor}}`, `latest`
  (default branch only), `sha-<short>` for every push
- Layer caching via GHCR `buildcache` tag (`mode=max`)

### 4. Startup-aware fast polling in /setup
- `maybeStartFastPoll(h)` reads `/health` response (`uptime_s`, `checks.*`)
- Shows a pulsing `startupBanner` if `!allOk && uptime < 120s`
- Starts a 5-second polling interval that stops when all checks pass or uptime ≥ 120s
- `load()` is monkey-patched to also call `maybeStartFastPoll` after each full refresh

### 5. Inline "Test LLM" button in /configure
- `testLlm()` calls `POST /api/test/llm` (existing endpoint), shows result inline in
  `#testResult` div: provider name, model ID, latency ms, or error message
- CSS states: `.ok` (green), `.err` (red), `.busy` (yellow spinner)
- No redirect — result appears next to the model ID field

### 6. First-launch setup overlay
- (See P1 description above — same feature, part of both improvement tracks)

### 7. Provider-aware model ID suggestions
- 6 `<datalist>` elements in `configure.html`, one per provider
  (cloudera, openai, azure, bedrock, anthropic, local)
- `updateModelDatalist(provider)` swaps the `list` attribute on `#LLM_MODEL_ID`
  to point at the matching datalist on provider selector change

### 8. .env download button
- `exportEnv()` fetches `/api/configure`, constructs `.env` file content
  (secrets shown as `<REDACTED>`), triggers browser download via `URL.createObjectURL`
- Button label: `⬛ Download .env` in the action bar

### 9. In-app log viewer at /setup
- Already implemented in a prior session; verified working
- `GET /api/logs` returns last N lines from `_MemoryHandler` ring buffer
- Displayed in a `<pre>` panel in the setup page, refreshed on each load

### 10. QR code popup in /setup
- `app/static/vendor/qrcode.min.js` — self-hosted davidshimjs QRCode.js (19,927 bytes)
- QR button in `setup.html` topbar: `toggleQr()` shows/hides `.qr-popup` div
- Generates QR on first open pointing to `window.location.origin + '/'`
- Self-hosted to remain usable in air-gapped / restricted-network CML environments
