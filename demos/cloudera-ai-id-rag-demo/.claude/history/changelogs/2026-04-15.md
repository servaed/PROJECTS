# Changelog — 2026-04-15

## Summary
Full-day session focused on fixing gabungan routing, improving retrieval quality, adding Azure
OpenAI support, upgrading the embeddings model, and a series of bug fixes uncovered during
a complete 36-question evaluation run across all 3 domains and 2 languages.

---

## Fixes

### Critical — Gabungan SQL leg always returned TIDAK_DAPAT_DIJAWAB
- **Root cause**: Combined questions ("apakah X sudah sesuai target kebijakan?") contain
  policy-comparison phrasing that confuses the SQL generator.
- **Fix**: Added `_extract_data_question()` in `answer_builder.py` — an LLM call that
  strips policy framing and returns a clean SQL-answerable question before SQL generation.

### Critical — Router misclassified gabungan as dokumen
- **Root cause 1**: Reasoning models return `<think>...</think>\ngabungan`; exact dict
  lookup on the raw string failed.
- **Root cause 2**: English gabungan questions missed all Indonesian regex patterns.
- **Fix**: 4-tier heuristic classifier in `router.py` with bilingual patterns + thinking-tag
  stripping. LLM fallback retained for truly ambiguous cases.

### High — Settings (LLM URL, API key) lost on app restart
- **Root cause**: pydantic-settings `env_file` only populates typed fields; generic keys
  mapped to `@property` accessors were ignored when loading from file.
- **Fix**: `_load_override_env()` pre-loads `data/.env.local` into `os.environ` before
  the `Settings()` singleton is constructed.

### High — Auto-restart after configure didn't work on Windows
- **Root cause**: uvicorn `--reload` reloader holds the TCP socket. Killing only the
  worker caused the reloader to immediately spawn a new old-code worker.
- **Fix**: Capture `parent_pid = os.getppid()`, kill entire parent tree via PowerShell
  `taskkill /F /PID {parent_pid} /T`, then spawn fresh uvicorn.

### Medium — SQL results displayed in scientific notation (2.368e+10)
- **Fix**: `to_markdown_table()` formats integer-valued floats with `:,` and true floats
  with `:.2f` before calling pandas `to_markdown()`.

### Medium — UnicodeEncodeError crashing Windows worker after hot-reload
- **Root cause**: `→` (U+2192) in `logger.info()` format strings; Windows cp1252 console
  can't encode it; crashes the logging thread, preventing clean hot-reload.
- **Fix**: Replaced all `→` with `->` in `router.py` log messages.

### Low — `<think>` blocks appearing in SQL output from reasoning models
- **Fix**: Added regex stripping of `<think>...</think>` in `query_generator.py` before
  SQL parsing.

## Quality Improvements

### Hybrid BM25 + FAISS retrieval
- Replaced pure FAISS with Reciprocal Rank Fusion of FAISS (semantic) + BM25 (keyword).
- BM25 catches entity names, branch codes, and numeric IDs that embedding models miss.
- Tuning: `_RRF_K=60`, `_FAISS_WEIGHT=0.6`, `_BM25_WEIGHT=0.4`.
- `rank-bm25>=0.2.2` added to `requirements.txt`.

### SQL schema semantic enrichment
- `_COLUMN_DESCRIPTIONS` dict in `metadata.py` adds human-readable descriptions to all
  40+ columns across 9 tables. Injected as inline SQL comments in the schema context.

### Dual-question retrieval for gabungan
- Two FAISS retrieval passes per combined question: one for the original (policy side),
  one for the data-extracted rephrase (quantitative side). Deduplicated by `(source, chunk)`.

### Conversation history depth
- `MAX_HISTORY_TURNS`: 3 → 5 (last 10 messages retained in prompt context).

### Embeddings model upgrade
- `multilingual-e5-base` (278M, 768-dim) → `multilingual-e5-large` (560M, 1024-dim).
- Better recall for short Bahasa Indonesia entity queries.
- Requires ~3 GB RAM; safe for 4+ GB Cloudera workspace profiles.
- Vector store rebuilt at 1024-dim with 63 chunks.
