## Session 1 — Streaming, Conversation History, Cloudera Brand UI

## Added
- `src/llm/base.py` — `stream_chat()` abstract method with non-streaming fallback
- `src/llm/inference_client.py` — `stream_chat()` using OpenAI streaming API
- `src/llm/prompts.py` — `SYSTEM_PROMPT_DATA`, `build_data_prompt()`, `_trim_history()`,
  `MAX_HISTORY_TURNS` constant; conversation history support in all prompt builders
- `src/orchestration/answer_builder.py` — `AnswerPrep` dataclass, `prepare_answer()`,
  `stream_synthesis()`, `finalize_answer()` (two-phase streaming pipeline)
- `app/ui.py` — full Cloudera brand theme (CSS, SVG logo, dark sidebar, mode badges,
  citation cards, custom error/status components)
- `.streamlit/config.toml` — Streamlit native theme (orange primary, dark text, port 8080)
- `data/sample_docs/prosedur_kyc_nasabah.txt` — KYC/APU-PPT banking compliance document

## Fixed
- SQL answer mode now uses `build_data_prompt()` instead of `build_document_prompt()`,
  giving the LLM the correct framing for structured data results
- `answer_question()` is backwards-compatible — still works non-streaming for tests

## Changed
- `app/main.py` — two-phase flow: spinner → `st.write_stream` → citations
- `app/ui.py` — `render_answer` split into `render_mode_badge` + `render_citations`;
  sidebar redesigned with dark Cloudera gradient + status panel
- Sample prompts updated to include KYC question aligned with new document
