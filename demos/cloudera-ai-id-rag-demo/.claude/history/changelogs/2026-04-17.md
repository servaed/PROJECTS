# Changelog — 2026-04-17

## P0: Embedded Cloudera CDP Stack (MinIO + Nessie + Trino + Iceberg)

### Architecture goal
Make the demo portable and self-contained — single Docker image with all data services
embedded, deployable on developer laptops and Cloudera AI Workbench (CML) without any
external dependencies. Each embedded service mirrors a real Cloudera CDP component.

### New connectors

- `src/connectors/trino_adapter.py` (new) — Trino Python client implementing the same
  interface as db_adapter's SQLAlchemy layer: `execute_read_query`, `get_table_names`,
  `get_table_schema`. Routes via Nessie REST catalog to Iceberg tables on MinIO.

- `src/connectors/ozone_adapter.py` (new) — boto3 S3 client for MinIO / Ozone S3GW.
  Returns relative `Path` objects (e.g. `Path("banking/file.txt")`) so `_infer_domain()`
  continues working without changes.

### Updated connectors

- `src/connectors/db_adapter.py` — rewritten as a factory. `QUERY_ENGINE=trino` dispatches
  to `trino_adapter`; default falls through to SQLAlchemy/SQLite. Added `get_engine()` public
  alias (used by `app/api.py` health check).

- `src/retrieval/document_loader.py` — `load_documents()` branches on `DOCS_STORAGE_TYPE`.
  `s3` → `OzoneAdapter` with `base_path=Path(".")` (S3 keys are already relative paths);
  `local` → `FilesAdapter` (unchanged).

### Dockerfile — multi-stage build

- Stage 1 (`infra`, debian:bookworm-slim): downloads MinIO binary, Nessie 0.83.2 JAR,
  Trino 455 tarball
- Stage 2 (python:3.11-slim): openjdk-17-jre-headless + curl + libgomp1 + Python deps
- Copies infra binaries from stage 1
- `COPY deployment/trino/etc /opt/trino/etc`
- ENV: `QUERY_ENGINE=trino`, `DOCS_STORAGE_TYPE=s3`, `JAVA_HOME=...`
- CMD: `["bash", "deployment/entrypoint.sh"]`

### deployment/entrypoint.sh (new)

Sequential startup with HTTP health-check loops:
1. MinIO :9000 → wait `/minio/health/live`
2. Nessie :19120 → wait `/api/v1/config`
3. Trino :8085 → wait `/v1/info` (100 × 3 s max)
4. `python deployment/seed_iceberg.py`
5. Build FAISS vector store if missing
6. `exec uvicorn app.api:app --host 0.0.0.0 --port $PORT`

### deployment/seed_iceberg.py (new)

- Creates MinIO buckets `rag-docs` and `rag-warehouse`
- Uploads all files from `data/sample_docs/` preserving `banking/`, `telco/`, `government/` prefixes
- `CREATE SCHEMA iceberg.demo WITH (location='s3://rag-warehouse/')`
- Drops and recreates all 9 tables as Iceberg tables (BIGINT, DOUBLE, VARCHAR, INTEGER)
- Inserts 148+ rows using multi-row VALUES

### deployment/trino/etc/catalog/iceberg.properties (new)

Iceberg connector pointing to Nessie REST catalog + MinIO:
```properties
connector.name=iceberg
iceberg.catalog.type=rest
iceberg.rest-catalog.uri=http://localhost:19120/iceberg/v1
iceberg.rest-catalog.warehouse=s3://rag-warehouse/
fs.native-s3.enabled=true
s3.endpoint=http://localhost:9000
s3.path-style-access=true
```
